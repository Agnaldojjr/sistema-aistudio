# Project Conventions

## Git Workflow
- Always create a new dedicated branch for major code changes.
- Branch name format should follow: `feature/[task-slug]` or `fix/[bug-slug]`.

## Preservação de Código e Dados (Mandatório)
- Nenhuma nova atualização ou script automatizado deve sobrescrever implementações já existentes ou apagar dados sem o consentimento prévio e explícito do usuário.
- Se houver riscos de perda de dados locais ou do Supabase, o usuário deve ser ativamente consultado antes da execução.
- Sempre buscar criar códigos limpos, modulares, bem documentados e sem duplicações de lógica ou dados desnecessários.
